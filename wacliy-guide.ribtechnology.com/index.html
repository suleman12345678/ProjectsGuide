About This Guide

This comprehensive guide will walk you through the essential operations of WaCliy, from initially logging into your account to provisioning new WhatsApp sessions via QR code linking.

Authentication

How to access your WaCliy portal.

Secure Login Process

WaCliy utilizes a highly secure authentication portal to protect your WhatsApp sessions. To access the dashboard, you must provide your registered credentials.

1
Navigate to the main application URL.
2
Enter your registered email address (e.g., rib@gmail.com).
3
Input your secure password. You can use the eye icon to toggle visibility.
4
Click the Sign In button to authenticate.
The WaCliy login portal featuring a dark layout with green accents.

Dashboard Overview

Navigating the main control center.

Your Command Center

Once authenticated, you will arrive at the main Dashboard. This interface provides an overview of your active sessions and quick access to core functionalities.

Navigation Breakdown

  • Sidebar Navigation: Access 'Sessions' or general 'WhatsApp' settings from the left panel.
  • Session List: View all your previously created sessions, their statuses (e.g., Disconnected/Active), and quick action buttons.
  • User Profile: Located at the bottom left, manage your account settings or access the Admin Panel.
The main dashboard showing navigation and session management.

Creating Sessions

Initialize a new WhatsApp instance.

Provisioning a New Instance

To link a new WhatsApp account, you first need to create a dedicated session container within WaCliy.

1
From the dashboard or sidebar, click the prominent Create New Session button.
2
A prompt will appear requesting a session name. Enter a recognizable identifier (e.g., "Support Line" or "Personal Account").
3
Click Create & Connect to initialize the instance.
Entering session details to provision a new WhatsApp instance.

Connecting Device (QR)

Linking your WhatsApp mobile app to WaCliy.

Establishing Secure Link

After creating a session, the system will generate a unique QR code. You must scan this code using the WhatsApp application on your mobile device to authorize the connection.

Time Sensitive: The generated QR code refreshes automatically every 30 seconds for security purposes. Ensure you scan it promptly.
1
Open WhatsApp on your primary mobile phone.
2
Navigate to Menu (Android) or Settings (iOS) and select Linked Devices.
3
Tap the Link a Device option. Authenticate if prompted by your phone.
4
Point your phone's camera at the QR code displayed on the WaCliy screen.
The standard connection screen waiting for QR code scan.

API Access

Programmatically control your WhatsApp sessions.

REST API Capabilities

WaCliy provides a robust HTTP API allowing developers to send messages, manage sessions, and automate workflows externally.

Authentication

Authentication is handled via Bearer tokens. Each session generates its own unique x-api-token which must be included in the headers of your HTTP requests.

  • Endpoint: POST /api/send
  • Format: JSON Payload
  • Built-in Guides: The dashboard includes direct integration templates for tools like n8n and command-line examples using cURL.
API configuration modal providing n8n and cURL templates.

Webhooks

Real-time event subscriptions.

Receive Incoming Messages

Configure webhooks to get real-time HTTP POST notifications whenever an event occurs on your WhatsApp session, such as receiving a new message.

1
From the session options (three dots), select Webhooks.
2
Enter your destination URL (e.g., your generic server endpoint or an n8n Webhook node URL).
3
When a message arrives, WaCliy sends a JSON payload containing the sender details, message body, session ID, and timestamp.
Defining a custom webhook URL for incoming messages.
